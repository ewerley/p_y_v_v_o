pyvvo.optimizers.pso.pso module¶

class pyvvo.optimizers.pso.pso.Pso(c1=1.0, c2=1.5, wmin=0.3, wmax=0.9)[source]¶

Bases: object

A Python class for the PSO optmizer

Variables:
  • c1 (float, by default 1.) – Learning (cognitive) parameter
  • c2 (float, by default 1.5) – Global learning (social) parameter
  • wmin (float, by default 0.3) – Minimum weight
  • wmax (float, by default 0.9) – Maximum weight
  • npart (int) – Number of particles
  • dim (int) – Problem’s dimension
  • x (numpy.ndarray, (npart, dim)) – Position vector of the population
  • v (numpy.ndarray, (npart, dim)) – Velocity vector of the population
  • w (float) – Velocity weight
  • xbest (numpy.ndarray, (npart, dim)) – Best position vector of each particle
  • xbest_fitness (numpy.ndarray, (npart, 1)) – Fitness value of xbest
  • gbest (numpy.ndarray, (npart, 1)) – Best position vector among all particles (global best)
  • gbest_fitness (numpy.ndarray, (itermax, 1)) – Fitness value of the global best
  • gbest_fitness – Fitness value of the global best at each iteration

Examples

Minimization of the Six Hump camel function. The function has a known optimal (minimum) at (0.0898,-0.7126) and (-0.0898,0.7126), with a function value of -1.0316,

>>> from pyvvo.optimizers.pso import Pso
>>> from pyvvo.optimizers.pso.report import fitness_plot
>>>
>>> # Function to be optimized
>>> def sim_hump_camel(x):
>>>     x1, x2 = x
>>>     f = (4 - 2.1*x1**2 + (x1**4)/3)*x1**2 + x1*x2 + (-4 + 4*x2**2)*x2**2
>>>     return f
>>>
>>> # Define a Pso object
>>> pso = Pso()
>>>
>>> # Optimize
>>> pso.optimize(
>>>     fitnessfunc=sim_hump_camel,
>>>     npart=10,
>>>     itermax=50,
>>>     xmin=[-3, -2],
>>>     xmax=[ 3,  2], report=True, show_progress=True
>>>     )
>>>
>>> # Print results
>>> print("Results:")
>>> print(' Found optimal x* at ({:.5g},{:.5g})'.format( pso.gbest[0], pso.gbest[1]))
>>> print(' F(x*)={:.5g}'.format(pso.gbest_fitness))
>>> fitness_plot(pso, subtitle='Six-hump Camel Function', ylabel='F(x1,x2)')
optimize(fitnessfunc, npart, xmin, xmax, xstep=None, direction='min', itermax=200, discretization=None, xclamp=False, vclamp=False, vmin=None, vmax=None, show_progress=False, report=False)[source]¶

Run the PSO optimizer

Parameters:
  • fitnessfunc (callable) – A callback function that takes a numpy.ndarray as argument and returns the corresponding fitness value.
  • npart (int) – Number of particles
  • itermax (int, by default 200) – Maximum number of iterations
  • xmin (numpy.ndarray, (1, dim)) – Lower limit of each variable of the position vector
  • xmax (numpy.ndarray, (1, dim)) – Upper limit of each variable of the position vector
  • vmin (numpy.ndarray, (1, dim), optional) – Lower limit of each dimension of the particle’s velocity
  • vmax (numpy.ndarray, (1, dim), optional) – Upper limit of each dimension of the particle’s velocity
  • xstep (numpy.ndarray, (1, dim), optional) – Step size of each (discrete) variable of the position vector. A None value indicates a continuous variable.
  • direction ({'min', 'max'}, by default 'min') – Optimization direction. If ‘min’, seeks for a minimum. If ‘max’, seeks for a maximum.
  • discretization ({'iter', 'mid-iter , 'end'}, by default None) – Discretization strategy. In the ‘iter’ case, the discrete variables are rounded to the next discrete value at every iteration. In the ‘mid-iter’ case, the discretization is enabled only after maxiter/2 iterations. In the ‘end’ strategy, the discretization is performed only after the final convergence.
  • xclamp (bool, by default False) – If True, each element of the position vector is forced to remain between the limits in xmin and xmax.
  • vclamp (bool, by default False) – If True, each element of the velocity vector is forced to remain between the limits in vmin and vmax.
  • report (bool, default False) – Print the PSO report, by default False
  • show_progress (bool, default False) – Enable the bar progress, by default False
pyvvo.optimizers.pso.pso.rand(d0, d1, ..., dn)¶

Random values in a given shape.

Note

This is a convenience function for users porting code from Matlab, and wraps random_sample. That function takes a tuple to specify the size of the output, which is consistent with other NumPy functions like numpy.zeros and numpy.ones.

Create an array of the given shape and populate it with random samples from a uniform distribution over [0, 1).

Parameters:d1, .., dn (d0,) – The dimensions of the returned array, must be non-negative. If no argument is given a single Python float is returned.
Returns:out – Random values.
Return type:ndarray, shape (d0, d1, ..., dn)

See also

random()

Examples

>>> np.random.rand(3,2)
array([[ 0.14022471,  0.96360618],  #random
       [ 0.37601032,  0.25528411],  #random
       [ 0.49313049,  0.94909878]]) #random
