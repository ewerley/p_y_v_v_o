A class to perform Voltage-Var Optimization in Distribution Systems

Variables:
  • vvofile (str) – Path to the main .vvo file with control definitions
  • sysfile (str, optional) – Path to the associated .dss file with network data. If not provided, a file with the same name as vvofile and a .dss extension will be sought.
  • grid (powergrid, default None) – Power system model represented as a Powergrid object. If should be provided in case sysfile is not given.
  • capvvocontrol (dict [str, dict]) –

    Capacitor control data. The keys str are the identifiers of the controlled capacitors, as in pyvvo.grid.capacitor. Each subkey is a dictionary with the following control properties:

    Maxkvar : float
    Maximum kVar of the bank (total of all phases)
    Numsteps : int
    Number of steps in the capacitor bank.
  • trvvocontrol (dict [str, dict]) –

    Transformer control data. The keys str are the identifiers of the controlled transformers, as in pyvvo.grid.transformer. Each internal dict should have the following control properties:

    MinTap : float
    Minimum per unit tap
    MaxTap : float
    Maximum per unit tap
    NumTaps : int
    Total number of taps between min and max tap
  • invertervvocontrol (dict [str, dict]) –

    Smart-inverter control data. The keys str are the identifiers of the controlled generators, as in pyvvo.grid.generator Each dict subkey should have the following control properties:

    MinPF : float
    Minimum power factor allowed at the generation point.
    MaxPF : float
    Maximum power factor allowed at the generation point.
    Maxkva : float
    Maximum kvA at the generation point.
  • __version__ (str) – Current version of the Vvo package.
  • optimizer – Reference to the external optimizer object.
  • lf (pyvvo.loadflow.Loadflow) – Reference to the internal loadflow object.
  • penalties (list [ str ] or dict) – A list of optimization penalties. See pyvvo.Vvo.optimize.
  • discretization (str, by default None) – Discretization strategy passed to the solver.
  • nc (int) – Number of controls.
  • ncap (int) – Number of capacitor controls.
  • ntr (int) – Number of transformer controls.
  • ninv (int) – Number of smart-inverter controls.
  • u (list [ float ]) – Actual values of the control variables, (nc, 1).
  • umin (list [ float ]) – Lower limit of each control variable, (nc, 1).
  • umax (list [ float ]) – Upper limit of each control variable, (nc, 1).
  • ustep (list [ float ]) – Step-size of each control variable of discrete type, (nc, 1).
  • uid (list [ list ], (nc, 4)) –

    For each control variable, the following variables are stored:

    type : {‘capacitor’, ‘transformer’, ‘generator’}
    Type of the controlled equipment.
    id\_ : str
    Identifier of the equipment

    name of the controlled parameter

optimize(goal='min_losses', method='PSO', penalties='v-load, kvar-cap, tap, fp-inv, kva-inv', reports='info, base, solver, opt', vnom_dv=1.0, **options)[source]¶

Perform the VVO optimization

Parameters:
  • goal ({'min_losses', 'min_dv'}, by default 'min_losses') – Optimization goal, whether Losses or Voltage-deviation minimization.
  • method ({'PSO'}, by default 'PSO') – Solution method (solver).
  • penalties (str or dict) –

    A list of commad-separated penalty types to apply in the Vvo optimization process. Each penalty is associated with a specific optimization constraint, as following:

    • v-load : minimum/maximum voltage magnitude of the loads
    • kvar-cap : minimum/maximum capacitors’ kVar
    • tap : transformers’ lower and upper tap limits
    • fp-inv : minimum/maximum power factor of the inverter connection
    • kva-inv : maximum kVa generated at the inverter connection

    The default penalty factor for all penalties is 1e5, except for the tap penalty, whith has a factor of 1e6. If a dict is provided, the penalty factors are taken from the dictionary values.

  • reports (list [ str ]) –

    A list of reports to print. The following types are available:

    • info : System information.
    • base_controls : Controls adjustments and objective function of the base-case
    • base_voltages : Base-case full voltage report
    • base : Synonym for 'base_controls, base_voltages'
    • base_loads : Base-case load report
    • base_vchart : Base-case voltages chart.
    • base_flows : Base-case power flow report.
    • solver_progress : Solver progress report
    • solver_output : Solver output report
    • solver : Synonym for ‘solver_progress, solver_progress’
    • opt_controls : Optimal controls and objective function
    • constraints : Constraints report
    • opt_voltages : Voltage report at the optimal solution
    • opt_flows : Power flow report at the optimal solution
    • fitness_plot : Objective function along the iteration process.
    • opt_vchart : Optimal voltage chart.
    • opt : Synonym for ‘opt_controls, constraints, opt_voltages, opt_flows, fitness_plot, opt_vchart’
  • vnom_dv (float, by default 1.) – Nominal voltage to assume in the minimum voltage-deviation goal.
  • **options – Aditional keyworded arguments passed to the solver.

A class to represent a bus-branch model of a power system.

Models for elements of power distribution and conversion: lines, transformers, regulators, capacitors and loads. All components are handled in a per-node basis, which enables the class to natively represent a natural (unbalanced) power network.

Variables:
  • circuit (dict [str, dict]) –

    Represents a multi-phase voltage source. The key str is the circuit (source) name. The internal subkeys (properties) can be as follows:

    bus1 : str
    Name of the source bus
    basekv : str
    rated Line-to-line kV
    pu : str
    Actual per unit operating voltage
    angle : str
    Base angle of the first phase (in degrees)
    MVAsc3 : str
    3-phase short circuit MVA
    MVAsc1 : str
    1-phase short circuit MVA
  • line (dict [str, dict]) –

    Represents a multi-phase line or cable. The key str is the line name. The subkeys of the internal dict are:

    bus1 : str
    Name of bus and the correponding nodes of terminal 1. The nodes should be provided in the following way: ‘bus.1.2.3’ If missing, the phase sequence ‘123’ is assumed.
    bus2 : str
    Name of bus and the correponding nodes of terminal 2
    Linecode : str, optional
    Name of the associated linecode object with impedance definitions
    Length : str
    Line length
    R1, X1, R0, X0 : str
    Positive and zero-sequence resistances, and reactances, in ohms per unit length.
    C1, C0 : str
    Positive and zero-sequence capacitances (nF / unit length)
    units : str
    Length units.
  • transformer (dict [str, dict]) –

    Represents a three- or single-phase transformer. The key is the element name. The main properties are:

    Phases : str
    Number of phases
    Windings : str
    Number of windings
    XHL : str
    Percent high-to-low reactance

    The following properties are read sequentially, for each winding:

    Wdg : str
    Winding integer identification
    Bus : str
    Bus (and nodes) to which the winding is conneted
    Conn : { ‘wye’, ‘delta’ }
    Connection type of the winding
    kV : str
    Rated kV
    kva : str
    Base kva rating of the winding
    %R : str
    Percent resistance on the rated kva base
    %Tap : str
    Per-unit tap of the winding

    Alternatively, the previous properties can be set for all windings at once, by using the following keys:

    Buses : str (array_like)
    Array of bus-nodes definitions
    Conns : str (array_like)
    Array of winding connections
    kVs : str (array_like)
    Array of rated kVs
    kvas : str
    Array of the rated kvas
    Taps : str
    Array of per-unit taps
    %Rs : str
    Array of percent resistances
  • regcontrol (dict [str, dict]) – Voltage regulator model (currenly not used by PYDSE)
  • load (dict [str, dict]) –

    Multi-phase load model. The key str is the load identification. The internal properties are:

    bus1 : str
    Name of the bus (and nodes) to which the load is connected.
    Phases : str (int like)
    Number of phases
    kV : str (float like)
    Nominal voltage.
    kW : str (float like)
    Nominal active power
    kvar : str (float like)
    Base kvar
    Model : {‘1’, ‘2’ , ‘5’}, where:
    1 is Constant power 2 is Constant admittance 5 is Constant current
    Conn : {‘wye’, ‘delta’}
    Connection type.
  • generator (dict [str, dict]) –

    Multi-phase generator. The key str is the generator identification. The internal properties are:

    bus1 : str
    Name of the bus (and nodes) to which the generator is connected.
    Phases : str (int like)
    Number of phases
    kV : str (float like)
    Nominal voltage.
    kW : str (float like)
    Nominal active power
    kvar : str (float like)

    1 is Constant power

  • busnodes (dict [str, dict]) – Associative array where the keys are the bus numbers, and the values are the numering indices associated to each node. This array defines the bus-node order of the voltage and current arrays.
  • orderbus (dict [str, `int]) – Array mapping bus names and the corresponding order position.
  • orderbusn (dict [str, `int]) – Array mapping bus names with nodes and the corresponding order position.
  • nbus (int) – Total number of buses
  • nnodes (int) – Total number of nodes
  • nbr (int) – Number of branches
  • nbrp (int) – Number of individual branches
  • dss (pyopendss.Dss object, default None) – It will have the Dss object when readfromdss is called.
  • sysname (str) – Name of the network, as read in the circuit attribute.
  • filepath (str) – Stores the filename the system data was read from.
  • sub_bus (str) – Name of the substation (source) bus, as read in circuit attribute.
  • sub_nph (int) – Number of phases in the substation

  • initv (numpy.array (nnodes,)) – Per-unit complex number assigned to each node , for initialization purposes (see calc_Vbase for more details)
  • trbus1 (list [str]) – List of transformers’ primary buses
  • trbus2 (list [str]) – List of transformers’ secondary buses
  • times (bool, default False) – Enables time measuring, if True
  • times_data (dict [str, float]) – Array where the keys are function markers and the values are the corresponding computing times (enabled if times is True)

Note

Example of valid bus identifiers:

300.1.2.3 (bus 300 with nodes 1, 2 and 3)

400.1.2 (bus 400 with nodes 1 and 2)

SRC (bus SRC, with nodes 1 to 3 depending on the number of phases)

busbranch()[source]¶

Build the bus-branch model of the network

The bus-branch representation of the system is defined in terms of nodes and edges, which are modeled in the busnodes and branches associative arrays. The representation of most elements of the system follows the OpenDSS standard.

Notes

  1. The following equipment are currently accepted:

    • multi-phase lines
    • multi-phase transformers
    • multi-phase capacitors (assumed as current injections)

    The parameters are included in its original states (ohms, siemens, and so forth). The series and shunt line parameters can be provided in terms of impedance matrices or sequency matrices.

  2. The order assumed for each bus and nodes is defined according to the branch reading sequence (lines then transformers). The orderbus and orderbusn hold the bus-node sequence for subsequent references.

calc_Vbase()[source]¶

Calculate the voltage bases for the system

The method is based on the analysis of the network graph (sweepfromroot). Starting from the root node, each edge is traversed and the terminal-bus voltage base is computed by multiplying the other bus by the transformation ratio of the branch. For lines, this ratio is unitary. For transformers, it is defined based on the transformer ratio.

class pyvvo.pyopendss.Dss[source]¶

Bases: object

An interface class to read OpenDSS data files

The .dss file format is essentially a text-based command language script which has models for several types of power delivery, conversion and source elements.

The new command is the basic .dss command PyDSE interprets, determining the type and properties of the elements to be added. The redirect and also batchedit keywords are also implemented. For a full reference of the OpenDSS command language syntax, please refer to [1].

Notes

Currently, the specific implementation of the .dss codes in PyDSE is as follows:

Circuit

Implements a multi-phase voltage source. PyDSE defines the substation

voltage and angle from the basekv and angle tags.
Line

Models a multi-phase lines or cables, as a model with series

and shunt capacitance. The parameters can be provided as symmetrical

components or impedance matrices (most usual), by assigning it to a

Linecode object.
Linecode Stores the impedance characteristics for lines and cables.
Transformer

Implements a three or single phase-transformer. connected either as

as ‘wye’ or ‘delta’.
Regcontrol

in PyDSE, a voltage regulator is considered as a transformer with

fixed ratio.
Capacitor Multi-phase shunt (grounded) capacitor.
Load

Multi-phase ‘wye’ or ‘delta’ connected load. PyDSE implements most

of the .dss load modes, such as constant power (type 1), constant

impedance (type 2) and constant current (type 5).
Generator Multi-phase ‘wye’ or ‘delta’, constant PQ generator.

See also

Powergrid

References

..[1] https://www.epri.com/pages/sa/opendss
